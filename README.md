# 多智能体临床资料汇总与文献参谋系统

> 基于 Spring AI 多智能体协作架构，实现临床医疗资料智能汇总、文献检索与辅助诊断

## 项目简介

本项目设计并实现了职责解耦的多智能体体系，用于临床资料汇总分析和医学文献参谋。通过多 Agent 协作完成异构系统数据集成、智能分析、辅助诊断建议等任务。

## 技术栈

| 技术 | 说明 |
|------|------|
| Spring WebFlux | 响应式编程模型 |
| Spring AI | 多智能体编排、RAG 检索增强 |
| Milvus | 医疗私有化知识库向量存储 |
| Spring AI | 函数调用、Prompt 工程 |
| Markdown 分块 | 智能文档分块策略 |

## 核心亮点

- **多智能体解耦架构**：资料汇总 Agent、文献检索 Agent、诊断分析 Agent 各司其职，支持独立扩容
- **轻量级状态机工作流**：基于 Spring WebFlux 自研，控制 Agent 调度流程
- **Markdown 智能分块**：提升医学文献检索准确率
- **异构系统集成**：完成 HIS、LIS、PACS 三大院内系统集成
- **私有化 RAG 知识库**：Spring AI + Milvus，完全私有化部署

## 系统架构

```
┌─────────────────────────────────────────┐
│           Workflow Engine                │
│      (Spring WebFlux 状态机)            │
└────────┬────────┬────────┬──────────────┘
         │        │        │
    ┌────▼──┐ ┌──▼─────┐ ┌▼──────────┐
    │ 资料汇总│ │ 文献检索│ │ 诊断分析   │
    │ Agent │ │ Agent  │ │ Agent     │
    └───┬───┘ └───┬────┘ └─────┬─────┘
        │         │             │
        └─────────┼─────────────┘
                  ▼
         ┌────────────────┐
         │   RAG 知识库    │
         │  (Milvus + AI) │
         └────────────────┘
```

## 模块说明

| 模块 | 说明 |
|------|------|
| agent-coordinator | 智能体调度引擎，状态机工作流 |
| agent-collector | 资料汇总智能体，集成 HIS/LIS/PACS |
| agent-retriever | 文献检索智能体，RAG 知识库检索 |
| agent-analyzer | 诊断分析智能体，辅助诊断建议 |
| common | 公共模块 |

## 快速开始

### 环境要求
- JDK 17+
- MySQL 8.0
- Redis 6.0+
- Milvus 2.x

### 启动步骤
1. 执行 `sql/init.sql` 初始化数据库
2. 修改配置文件中的数据库、Milvus 地址
3. 启动 agent-coordinator（含调度引擎）
4. 各 Agent 可独立部署启动
